Python + Flask

Python allows you to serve a web application dynamically via the Flask web framework. Flask is a set of libraries designed to minimize static configurations and code repetition through the use of Werkzeug WSGI and Jinja. Flask is designed to be nimble and highly configurable. This page will discuss how Flask was implemented for this project, but to explore Flask in more detail and see to if it's right for your next project, please click the below.

File Organization

For Flask to work, we need to build the /templates and /static directories. The /templates folder houses the project's HTML pages and the /static folder stores elements used by the html pages such as CSS, images and JASON scripts, each organized into their respective folders. For this project, however, we have implemented Blueprint and so /templates folder only contains base.html, which is the template we use to build all our webpages. base.html allows us to build page elements once and apply it without repeating our code (i.e. navigation) using Jinja.

The entire render begins from /app/__init__.py which initializes the /simple_pages/__init__.py. Blueprint then renders each of the pages using the def show(page) method and the HTML files in /simple_pages/templates. Testing the Code

Testing code is a major component of application development and an element we could hardly forget to implement here. Our testing is executed automatically and activated by the Git Workflow actions. As you can see in the screenshot, PyTest is installed as part of the Python 3.8 dependencies list, then run with the pytest -vv command. The tests are relatively simple and looks to see if the pages were rendered correctly by Flask. They are stored in the /tests folder and the test results can be reviewed on the Github Actions page. This is a great tool when you find your web pages are not being served as it should. To learn more about PyTest, please click on the blue link.
