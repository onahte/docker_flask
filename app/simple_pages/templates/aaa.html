Arrange. Act. Assert.

Arrange, Act, Assert or AAA is a testing paradigm to structure code testing. Testing is a critical step in the development cycle by providing empirical feedback of our code and preventing dangerous code from being integrated into the main code base. Following AAA helps organize the test and aids in yielding a successful testing phase. This page is an overview of the AAA pattern and how it was used in this project.

To employ the AAA model...

1. Arrange : Arrange or set up the test case, making sure that all the pieces needed for testing are in place. For instance, if the application relies on a database, a sample database needs to be prepared.
2. Act : Activate the code to elicit a response. For instance, if the application has a method to search a database, that method is called so that we can observe the outcome.
3. Assert : Predefine the expected outcome. For instance, the activated method should return the database search result. If the result matches the assertion, our code has passed.

Unit Testing with AAA

AAA is a great tool for unit testing because the AAA model is built for testing application components (or units). The idea behind unit testing is that for the application to function as a whole, each unit much function as designed. By employing AAA, we can automate the unit tests as we have done in this project - just as we automated deployment, our Github Workflow files deploy our unit tests. For a refresher on how we did this, please go back to the CI/CD page or click on the blue link.

Now, unit testing is only useful if it yields actionable information. AAA helps with this by organizing how we test and explicating specific assertions that the test must meet. In doing so, AAA helps us to work methodically and follow a workflow:

Create test case -> Review/Rework -> Baseline -> Execute test cases

By working in this manner, the entire process of development becomes agile and easier to make sure our final application is of high quality. It is easier to locate mistakes, document the progress and adapt our code quickly.

AAA Applied

Here we have a screenshot of the method that tests our addition instance. As you can see, it employs the AAA model, which also acted as a framework to build this test method. The Arrange step informs us to build the test case. Here, we would like to see if the method can create an Division instance. So, we have prepped the test by creating a tuple with two integers. The Act step informs us to perform the instantiation, which we do and save as calculation. The tuple is passed into the instance as the instance property. The final step, Assert informs us to test if the instantiation has been successful, so we pass in calculation and the class it belongs to which, in this case is Division into the assertion call isinstance. From this example, it's evident that following the AAA model makes testing simpler.
