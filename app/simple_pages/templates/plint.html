Pylint

While AAA gives us a model to test our code functionality, Pylint is a tool that checks for errors in our Python code. Configurable and easy to deploy, Pylint helps teams develop uniform code that adheres to the groups' standards.

Pylint Deployment

PyTest runs all our tests, including Pylint. As mentioned in our Python+Flask page, our Git Workflow files automates the installation and deployment of PyTest and Pylint. As you can see in our requirements.txt, we explicate the applications and tools we want to automatically download and install. By listing pytest-pylint, Pylint is installed as part of our automated pipeline. To review Git Workflow YAML, please navigate to the "Testing the Code" section of the Python+Flaks page or click on the blue link.

Configuring Pylint

Pylint is designed to be very easy to configure. By editing the .pylintrc library file, we can configure the Python version, what errors to show and much more. We can even configure a list of bad variable names the development team would like to avoid. This may possibly be a senior engineer's or tech lead's best friend as it can make code review less frustrating by enforcing some code etiquettes. The file is commented with instructions in detail, so it is very easy to navigate.
